## 翻译策略

### 1. 保留内容
- 代码块和代码示例
- Markdown 标题结构（# ## ###）
- 链接和图片引用
- 徽章（badges）
- 表格结构

### 2. 翻译内容
- 所有描述性文本
- 文档正文
- 提示和技巧内容

### 3. 格式
- 使用中文标点符号（，。：；？！""）
- 保持标题层级
- 保持代码块语言标识

### 4. 翻译顺序建议
1. 先翻译核心 best-practice 文档（理解概念）
2. 再翻译 implementation 文档（实践）
3. tips 和 reports（深入理解）
4. 其他辅助文档

## 实现方式
由于文件数量较多（50+个markdown文件），建议：
1. 分批翻译，每次处理一个目录
2. 使用 AI 辅助翻译保持一致性
3. 在 TRANSLATION.md 中记录翻译进度

## 注意事项
- 某些链接指向英文文档（如 code.claude.com/docs），保留英文
- GitHub 标签徽章保留原样
- 视频和外部资源链接保留英文
